# Dogcraft-AFK

A comprehensive AFK detection and management plugin for Paper 1.21+ servers. Designed to detect and prevent common AFK farming tricks while providing staff notifications, cross-server messaging, and LuckPerms integration.

## Features

### Two-Phase AFK System
Players go through two stages of inactivity:

| Phase | Default | What Happens |
|-------|---------|--------------|
| **AFK** | 2 minutes | Marked as AFK, suffix applied, stats frozen, staff notified |
| **Kick** | 7 minutes | Kicked from the server |

A warning countdown appears in chat before the kick, and an action bar message is shown throughout the AFK phase.

### Anti-AFK Trick Detection
The plugin detects and prevents common methods players use to avoid AFK detection:

- **Minecart loops** -- Tracks the last 10 unique chunks visited during vehicle rides. If the player loops back through chunks already in their history, the movement is ignored.
- **Bubble columns** -- Same chunk-based loop detection applied when the player is in a bubble column.
- **Water streams** -- Flowing water at the player's feet is treated as passive movement and routed through loop detection.
- **Piston pushers** -- If a player's position changes but their camera angle (yaw/pitch) stays identical, the movement is classified as passive. Real players almost always move their camera when walking.
- **Auto-clickers** -- Tracks the last 20 interact event timestamps and computes the standard deviation of intervals. If the deviation falls below a configurable threshold (default 50ms), the timing is flagged as unnaturally uniform.

When a trick is detected, staff with the `afk.notify` permission are notified. Trick notifications only fire once per player until they resume normal activity.

### Statistic Freezing
While AFK, the player's `PLAY_ONE_MINUTE` statistic is frozen by snapshotting the value when they enter AFK and resetting it every second. This prevents playtime from accumulating while idle.

### Clickable AFK Confirmation
When a player enters AFK, they receive a clickable chat message:

> You are now AFK. **[Click here to confirm you're not AFK]**

Clicking runs `/afk-confirm` to immediately clear AFK status.

### LuckPerms Integration (Optional)
If LuckPerms is installed, the plugin adds a configurable suffix (default: ` &7[AFK]`) to the player's LP data when they go AFK. This automatically appears in tab lists, chat, and any other plugin reading LuckPerms metadata. The suffix is:

- Applied when entering AFK
- Removed when returning from AFK, being kicked, quitting, or switching servers
- Cleaned up on join in case a stale suffix persists from a previous session

LuckPerms is a soft dependency -- the plugin works without it.

### BungeeCord Cross-Server Messaging (Optional)
When enabled, all staff notifications (AFK entered, returned, auto-clicker alerts, trick attempts) are forwarded to all servers in the BungeeCord network via the `BungeeCord` plugin messaging channel.

Additionally, a custom `dogcraft:afk` channel sends structured status updates using the AFK player as the sender:

| Field | Type | Description |
|-------|------|-------------|
| UUID | UTF string | The player's UUID |
| AFK | boolean | `true` = entered AFK, `false` = returned |

This allows proxy plugins to react to AFK state changes (e.g. forwarding to Discord, updating a global tab list).

### Server Identity
If a `server_id.conf` file exists in the server root (generated by another plugin), the `name` property is used as the server name in cross-server messages. Falls back to the config value if the file is absent.

```properties
#Auto-generated server identity - do not edit
name=s1
uuid=fc99fe8a-fefd-40bf-a123-931506d3cf78
```

## Commands

| Command | Description |
|---------|-------------|
| `/afk` | Toggle AFK status. If not AFK, manually marks you as AFK. If already AFK, returns you. |
| `/afk-confirm` | Confirm you are not AFK. Only works while in AFK state. |

## Permissions

| Permission | Default | Description |
|------------|---------|-------------|
| `afk.notify` | op | Receive staff notifications (AFK status changes, auto-clicker alerts, trick attempts) |
| `afk.exempt` | op | Completely exempt from AFK detection and kick |

## Configuration

All messages use [MiniMessage](https://docs.advntr.dev/minimessage/format.html) format (Adventure API), including the LuckPerms suffix.

```yaml
afk:
  afk-timeout: 120      # seconds before marked AFK (2 minutes)
  kick-timeout: 420     # seconds before kicked (7 minutes)
  # All messages use MiniMessage format: https://docs.advntr.dev/minimessage/format.html
  afk-message: "<gray>{player} is now AFK."
  return-message: "<gray>{player} is no longer AFK."
  kick-message: "<red>You have been kicked for being AFK too long!"
  warning-message: "<yellow>You will be kicked in <red>{seconds} <yellow>seconds for being AFK!"
  actionbar-message: "<yellow>You are AFK. Kick in <red>{seconds}s"
  afk-notice: "<yellow>You are now AFK. "
  afk-click-text: "<green><bold>[Click here to confirm you're not AFK]"
  warning-threshold: 30  # warn X seconds before kick
  freeze-stats: true     # pause PLAY_ONE_MINUTE stat while AFK

  luckperms-suffix:
    enabled: true
    suffix: " <gray>[AFK]"
    priority: 100         # higher priority overrides other suffixes

  bungee:
    enabled: false
    server-name: "Survival"       # fallback if server_id.conf is absent
    network-format: "<dark_gray>[{server}]</dark_gray> {message}"

  interact-detection:
    sample-size: 20           # number of interact events to analyse
    max-deviation-ms: 50      # std dev threshold -- below this is suspiciously uniform
    notify-message: "<red>[AFK] <yellow>{player} <red>may be using an auto-clicker!"

  trick-notify-message: "<red>[AFK] <yellow>{player} <red>is potentially trying to trick the AFK kicker!"
```

### Placeholders

| Placeholder | Used In | Value |
|-------------|---------|-------|
| `{player}` | Most messages | Player's name |
| `{seconds}` | Warning, action bar | Seconds until kick |
| `{server}` | Network format | Server name |
| `{message}` | Network format | The original message |

## Event Handling

All event listeners run at `MONITOR` priority and do **not** use `ignoreCancelled`. This means cancelled events (e.g. chat intercepted by a filter plugin, interact from a GUI click) still count as player activity. A cancelled action still proves someone is at the keyboard.

### Activity Events
These events reset the AFK timer:

| Event | Notes |
|-------|-------|
| `PlayerMoveEvent` | Only active movement (see below) |
| `AsyncChatEvent` | Modern Paper chat event |
| `PlayerCommandPreprocessEvent` | Any command attempt |
| `PlayerInteractEvent` | Also tracked for auto-clicker detection |
| `PlayerJoinEvent` | Also cleans up stale AFK suffix |

### Movement Classification

```
Position unchanged + rotation unchanged  -->  Ignored (no activity)
In vehicle / bubble column / water       -->  Passive (chunk loop detection)
Position changed + rotation unchanged    -->  Passive (piston/external force)
Position changed + rotation changed      -->  Active (resets AFK timer)
Rotation only changed                    -->  Active (resets AFK timer)
```

## Building

Requires Java 21 and Maven.

```bash
mvn clean package
```

The output jar is in `target/`.
